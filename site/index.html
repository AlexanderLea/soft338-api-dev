This API allows a user to be created, and then job applications associated with that user. When an application is created or updated, the API checks if the postcode entered is valid or not, and either rejects the operation if it's invalid, or continues if valid.

Getting Started

Authentication

In order to use this API, requests must be authenticated using an API key. To get an API key, you must submit a POST request to the /users endpoint. The API will respond with your API key.

To then use other resources, the API key must be included in the header of the request in the following format:

  • Header: Application-ApiKey
  • Value: [key value from previous POST request]

Errors

  • 400 Bad Request
      There was an error with the request data provided
  • 401 Unauthorized
      The API key is either not provided, or is incorrect
  • 404 Not Found
      The resource does not exist
  • 405 Method Not Allowed
      The utilised HTTP method is not allowed for this endpoint
  • 422 Unprocessable Entity
      The data provided is invalid (perhaps the postcode is invalid)

Description

Lists all users in the system. Must be authenticated in order to access this resource.

Response
